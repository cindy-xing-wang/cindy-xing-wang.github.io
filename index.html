The first time I setup a Hexo blog was about 3 years ago. I was following a blogger’s instructions copying and pasting his code without knowing what I was doing. I finally reached a goal– having my own hexo blog with default settings. I did not continue writing anything in the blog. Because I have nothing to write about programming at that time. I never thought I would shift my career to be a developer until this year. I went to polytech to study programming.I find myself have great passion on coding. Interestingly, I looked back to the same blogger’s instructions recently after studying programming for a year. I totally understand those code now. I am thinking maybe I can write something now. So why not start from building my own blog…

Quick Start

Hexo
When you create a repo on github, do remember use the same name as your github owner’s name. For example, my github owner’s name is ‘cindy-xing-wang’, my repo name has to be ‘cindy-xing-wang.github.io’. If you do not follow the rule, the deployed blog will not be able to style correctly.

About themes

The first thing to notice is that the lastest version (5.0.0) of Hexo has nothing (except .gitkeep which is an empty file) under themes’s fold. If you want to see the file structure, you can type this command in the terminal.

It will build a public folder and the default theme ‘landscape’ can be found there.
After built the blog, I browserd some other themes in Hexo themes. I decided to use theme Stun as it is still under maintainance and the documentation is really good.

More info:

  1. Try to bulid Hexo on macOS as some themes may not working on windows.
  2. I attempted to clone different themes. Some were not displaying the way I was exepcting. During the process, I happened to know Hugo which is another blog buliding framework written in Go language. I haven’t tried it yet. If you want to find a easier way to build a blog maybe this is the way to go.
